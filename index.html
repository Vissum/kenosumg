Términos de SQL Server

¿Qué es SQL Server?

SQL Server es un potente sistema de gestión de bases de datos relacionales desarrollado por Microsoft. Utiliza el lenguaje SQL estándar para almacenar, manipular y recuperar datos. Ofrece características robustas de seguridad, alta disponibilidad y escalabilidad. Incluye motores de bases de datos para escenarios relacionales, OLAP y streaming de datos. Además, proporciona herramientas de inteligencia empresarial para integración de datos, análisis y reportes. SQL Server es ampliamente adoptado en entornos empresariales para alojar aplicaciones de bases de datos críticas, data warehousing y análisis de datos, gracias a sus capacidades de administración y monitoreo integradas.

Para qué se utilizan las System Databases

  • Master:

    Propósito: La base de datos "master" es una de las bases de datos del sistema más importantes en SQL Server.

    Función:

    • Almacena la información sobre todas las bases de datos creadas en una instancia de SQL Server.
    • Contiene la información de inicio de sesión (logins) y roles de servidor.
    • Mantiene la información de configuración del servidor, como las rutas de los archivos de datos y registros, opciones de configuración del servidor, etc.
    • Es esencial para iniciar y ejecutar correctamente el servidor de SQL Server.

  • Model:

    Propósito: La base de datos "model" se utiliza como plantilla (template) cuando se crea una nueva base de datos en SQL Server.

    Función:

    • Contiene la estructura predeterminada de todas las nuevas bases de datos que se crean en el servidor.
    • Cualquier cambio que se realice en la base de datos "model" se aplicará automáticamente a las nuevas bases de datos que se creen después.
    • Se usa para configurar parámetros de base de datos por defecto, como el tamaño de los archivos de datos y de registro.

  • Msdb:

    Propósito: La base de datos "msdb" es una base de datos de sistema que se utiliza principalmente para administrar tareas y servicios relacionados con SQL Server.

    Función:

    • Almacena información sobre copias de seguridad (backups), mantenimiento de bases de datos, historial de trabajos de SQL Server Agent, etc.
    • Contiene información sobre alertas y operadores para notificaciones del sistema.
    • Utilizada por SQL Server Agent para programar y ejecutar tareas, como copias de seguridad, indexaciones, ejecución de scripts, etc.

  • Tempdb:

    Propósito: La base de datos "tempdb" es una base de datos temporal utilizada para almacenar datos temporales, como tablas temporales, variables de tabla y otros objetos temporales.

    Función:

    • Almacena datos temporales creados durante la ejecución de consultas, procedimientos almacenados, funciones y otros procesos.
    • Se utiliza para ordenar y agrupar datos temporalmente cuando se ejecutan operaciones de consulta complejas.
    • Se recrea cada vez que se reinicia SQL Server, por lo que cualquier información almacenada en "tempdb" es transitoria y no persistente.

Roles en los servidores

  • bulkadmin:

    Privilegios:

    • Puede realizar operaciones de bulk import (bulk insert).
    • Puede importar grandes volúmenes de datos desde archivos en el sistema de archivos del servidor hacia tablas en una base de datos.

  • dbcreator:

    Privilegios:

    • Puede crear, modificar y eliminar bases de datos en el servidor.
    • Tiene permisos para realizar tareas de administración de bases de datos, como cambiar propiedades de bases de datos, definir archivos y grupos de archivos, y configurar opciones de bases de datos.

  • diskadmin:

    Privilegios:

    • Puede administrar archivos de datos y registros en el sistema de archivos del servidor.
    • Tiene permisos para agregar o quitar archivos de datos y registros a las bases de datos, y para cambiar las ubicaciones físicas de los archivos.

  • processadmin:

    Privilegios:

    • Puede detener procesos de SQL Server.
    • Tiene permisos para monitorear y finalizar los procesos de usuario que se están ejecutando en el servidor.

  • public:

    Privilegios:

    • Es un rol predefinido al que se le otorgan ciertos permisos básicos en todas las bases de datos.
    • Los usuarios que pertenecen al rol "public" tienen acceso predeterminado a ciertos objetos y funciones dentro de las bases de datos, como vistas del sistema y procedimientos almacenados del sistema.

  • securityadmin:

    Privilegios:

    • Puede administrar cuentas de inicio de sesión y roles de servidor.
    • Tiene permisos para agregar y quitar inicios de sesión, cambiar propiedades de inicio de sesión, asignar roles de servidor, entre otros.

  • serveradmin:

    Privilegios:

    • Puede realizar tareas de administración a nivel de servidor.
    • Tiene permisos para configurar opciones de servidor, detener y reiniciar el servidor, y cambiar configuraciones globales del servidor.

  • setupadmin:

    Privilegios:

    • Puede administrar la instalación de SQL Server.
    • Tiene permisos para agregar y quitar cuentas de servicios, configurar opciones de instalación y administrar las propiedades de la instancia.

  • sysadmin:

    Privilegios:

    • Este es el rol de servidor más alto y poderoso en SQL Server.
    • Tiene todos los permisos posibles a nivel de servidor y base de datos.
    • Los usuarios que pertenecen a este rol tienen control total sobre todos los recursos de SQL Server, incluyendo la capacidad de agregar o eliminar usuarios, bases de datos, roles, procedimientos almacenados, tablas, entre otros.

¿Qué significa la función Replication?

  • Balanceo de carga:

    Replicar datos para distribuir la carga entre varios servidores SQL Server, permitiendo que las consultas se ejecuten en servidores locales en lugar de en un servidor central.

  • Alta disponibilidad:

    Crear copias redundantes de bases de datos para asegurar que los datos estén disponibles en caso de fallo de hardware o de servidor. Esto incluye escenarios como la replicación de alta disponibilidad (AlwaysOn Availability Groups).

  • Consolidación de datos:

    Recopilar datos de varias ubicaciones geográficas o sucursales en una ubicación central, manteniendo copias actualizadas de los datos en tiempo real.

  • Integración de datos:

    Sincronizar datos entre diferentes sistemas o bases de datos, permitiendo que los datos estén disponibles para aplicaciones y procesos en diferentes plataformas.

  • Reporting y análisis:

    Crear copias de bases de datos para ejecutar informes o análisis de datos sin afectar el rendimiento de los sistemas de producción.

  • Replicación por instantáneas (Snapshot Replication):

    Toma una instantánea de la base de datos en un momento específico y la replica en otros servidores. Es útil para situaciones en las que los datos no cambian con frecuencia.

  • Replicación transaccional (Transactional Replication):

    Replica cambios de datos en tiempo real o casi en tiempo real desde una base de datos a otra. Es útil para aplicaciones que necesitan sincronización en tiempo real.

  • Replicación de mezcla (Merge Replication):

    Permite cambios bidireccionales entre las bases de datos, lo que es útil en escenarios donde las bases de datos pueden cambiar en múltiples ubicaciones y necesitan fusionarse.

  • Replicación peer-to-peer (Peer-to-Peer Replication):

    Cada base de datos es tanto un editor como un suscriptor, lo que significa que los cambios realizados en cualquier ubicación se propagan a otras. Es útil para cargas de trabajo que requieren alta disponibilidad y escalabilidad.

¿Cuál es la función de SQL Profiler?

  • Captura de Eventos:

    SQL Server Profiler captura una amplia variedad de eventos que ocurren en una instancia de SQL Server. Estos eventos pueden incluir consultas ejecutadas, cambios de esquema, errores, inicio y cierre de sesiones, entre otros.

  • Análisis en Tiempo Real:

    Permite ver en tiempo real las consultas y acciones que se están ejecutando en una base de datos de SQL Server. Esto es útil para monitorear el rendimiento y la actividad del servidor en tiempo real.

  • Creación de Perfiles Personalizados:

    Los usuarios pueden crear perfiles personalizados para capturar solo los eventos específicos que les interesan. Esto ayuda a reducir el ruido y enfocarse en los eventos relevantes.

  • Identificación de Cuellos de Botella:

    Al monitorear las consultas y eventos en tiempo real, los administradores pueden identificar fácilmente las consultas lentas, los bloqueos y otros problemas de rendimiento.

  • Optimización de Consultas:

    Al analizar las consultas capturadas, los desarrolladores pueden identificar oportunidades para optimizar consultas y mejorar el rendimiento de las aplicaciones.

  • Diagnóstico de Problemas:

    Facilita la detección y resolución de problemas en una base de datos de SQL Server. Por ejemplo, puede ayudar a identificar consultas que causan bloqueos o errores.

  • Auditoría y Seguridad:

    SQL Server Profiler también se puede utilizar para auditar acciones realizadas en una base de datos, como cambios en datos sensibles o acceso a recursos críticos.

  • Generación de Informes y Análisis:

    Los datos capturados por SQL Server Profiler pueden ser guardados en archivos o tablas de base de datos para su posterior análisis. También se pueden generar informes para revisión y análisis.
